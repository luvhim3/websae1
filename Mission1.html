    CONTEXTE : L'entreprise Kdou fait appel à notre équipe pour améliorer leur image "éco-friendly". Pour cela, il faudra prendre en charge l'évolution de l'organisation et de leur positionnement sur le marché de l'alimentaire tout en respectant une démarche "éco-friendly"

Dans cette première mission, le but premier est d'analyser les tables et comprendre leur fonctionnement. Pour cela, nous avons répondu à de nombreuses tâches, les voici :

    1/ Il faut identifier quelles sont les colonnes pouvant correspondre à des clés primaires.
  • FOURNISSEUR : n° fournisseur
  • PRODUIT : réf produit
  • CATEGORIE : code catégorie
  • CLIENTELE : code client
  • COMMANDE : n° commande
  • DETAILS COMMANDE : couple de clés étrangères qui forment une clé primaire -> réf produit + n° commande

BILAN : Après avoir parcouru cette Base de Données, nous pouvons constater qu'il y a certain problème qui peut nous empêcher une bonne prise en main de la Base de Données. Tout d'abord dans la question 3.h il nous est demandé de chercher une personne au nom de "Mikal Pavbo". Cependant, dans notre table son nom est enregistré sous le nom de "Mikal Torin" et cela pose un problème dans notre recherche, car l'on ne peut pas savoir si c'est bien "Mikal Pavbo" et pouvoir répondre à cette demande de recherche. De plus dans la table fournisseur le nom des pays son écrit de manière différente par exemple : les "États-Unis" possèdent 3 façons d'écrire "Etats-Unis", "Etats Unis", "USA" avec ces différences la recherche devient plus compliqués d'autant plus qu'il y a d'autre pays dans ce cas-là. Pour finir l'utilisation d'Excel est plutôt limité, car l'on n'a pas la possibilité de faire des requêtes précise et plus approfondi, c'est pour cela que l'utilisation de Access est plus appropriés dans la démarche d'amélioration de l'entreprise Kdou
